# 不提交密钥的规则

本规则定义了跑团网站项目中关于敏感信息的安全要求。

## 禁止提交的内容

以下敏感信息**绝对禁止**提交到代码仓库：

### 1. 认证凭证
- 数据库用户名和密码
- API密钥和密钥
- JWT密钥或其他签名密钥
- OAuth客户端密钥
- 服务账号凭据

### 2. 私钥和证书
- SSL/TLS私钥
- SSH私钥
- PGP私钥
- 代码签名证书

### 3. 第三方服务凭证
- 云服务商（AWS、Azure、GCP）访问密钥
- 支付网关API密钥
- 短信/邮件服务API密钥
- 第三方登录API密钥

### 4. 配置文件
- 包含真实凭据的配置文件
- .env 文件
- 包含密码的配置模板

## 正确做法

### 1. 使用环境变量
所有敏感配置应通过环境变量管理：
```javascript
// 错误 - 硬编码密钥
const SECRET_KEY = "my-secret-key-12345";

// 正确 - 使用环境变量
const SECRET_KEY = process.env.SECRET_KEY;
```

### 2. 使用配置文件模板
提供配置文件的模板版本，但不包含真实值：
```
# 错误 - .env
DB_PASSWORD=my-real-password

# 正确 - .env.example
DB_PASSWORD=your-password-here
```

### 3. 添加到 .gitignore
确保以下文件被忽略：
```
.env
.env.local
*.pem
*.key
credentials.json
secrets.yaml
```

### 4. 使用密钥管理服务
- 生产环境使用云服务商的密钥管理服务
- 本地开发使用本地密钥库

## 跑团网站项目特定要求

### 必需的敏感配置
跑团网站项目需要保护以下配置：
- 数据库连接字符串
- 会话密钥
- WebSocket连接密钥
- 第三方服务API密钥（如有）

### 开发环境
- 使用单独的测试数据库
- 使用假的测试账户
- 不使用真实的生产凭据

### 生产环境
- 通过安全渠道获取凭据
- 定期轮换密钥
- 记录谁有权限访问密钥

## 违规处理

如果发现敏感信息被提交：

1. **立即行动**
   - 撤销暴露的密钥
   - 轮换所有相关凭据
   - 检查是否有未授权访问

2. **修复代码**
   - 删除敏感信息
   - 改为使用环境变量
   - 强制推送到远程仓库

3. **总结教训**
   - 记录事件
   - 改进流程防止再次发生

## 验证步骤

在提交代码前，检查以下内容：

- [ ] 没有硬编码的密码或密钥
- [ ] .env 文件未被跟踪
- [ ] 配置文件模板不包含真实值
- [ ] 日志中没有敏感信息
- [ ] 错误消息不泄露系统细节
