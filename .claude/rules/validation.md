# 代码验证规则

本规则定义了跑团网站项目开发中的代码验证要求。

## 1. 前端代码验证

### 1.1 HTML/CSS验证
- [ ] 所有HTML标签正确闭合
- [ ] CSS属性使用正确
- [ ] 无过时的HTML标签
- [ ] 响应式布局在各断点正常工作

### 1.2 JavaScript验证
- [ ] 无语法错误
- [ ] 无未使用的变量
- [ ] 无全局变量污染
- [ ] 错误处理完善
- [ ] 无内存泄漏

### 1.3 前端特定检查
- [ ] 表单验证完整
- [ ] 用户输入正确转义（防XSS）
- [ ] API调用有超时处理
- [ ] WebSocket连接有重连机制

## 2. 后端API验证

### 2.1 请求验证
- [ ] 所有输入参数进行验证
- [ ] 参数类型正确
- [ ] 必填参数已检查
- [ ] 参数长度限制

### 2.2 响应验证
- [ ] HTTP状态码正确
- [ ] 响应格式一致
- [ ] 错误消息有意义
- [ ] 无敏感信息泄露

### 2.3 认证授权验证
- [ ] 受保护的接口验证Token
- [ ] 权限检查正确
- [ ] Session管理安全

### 2.4 API特定检查
- [ ] 速率限制已实现
- [ ] CORS配置正确
- [ ] CSRF防护到位
- [ ] 日志不包含敏感信息

## 3. 数据库验证

### 3.1 查询验证
- [ ] 使用参数化查询（防SQL注入）
- [ ] 无SQL字符串拼接
- [ ] 查询有适当的索引
- [ ] 避免SELECT *

### 3.2 数据验证
- [ ] 数据类型正确
- [ ] 必填字段已验证
- [ ] 数据长度限制
- [ ] 唯一性约束已验证

### 3.3 事务验证
- [ ] 事务使用正确
- [ ] 错误回滚正常
- [ ] 无死锁风险

## 4. 安全验证

### 4.1 认证安全
- [ ] 密码使用强哈希算法
- [ ] 密码长度符合要求
- [ ] 登录尝试限制
- [ ] 会话超时合理

### 4.2 数据安全
- [ ] 敏感数据加密存储
- [ ] 敏感数据传输加密
- [ ] 无硬编码密码
- [ ] API密钥不暴露

### 4.3 输入安全
- [ ] 用户输入验证
- [ ] 文件上传安全检查
- [ ] URL参数验证
- [ ] JSON输入验证

## 5. 性能验证

### 5.1 前端性能
- [ ] 无阻塞渲染的脚本
- [ ] 图片已优化
- [ ] 资源已压缩
- [ ] 缓存策略合理

### 5.2 后端性能
- [ ] 数据库查询优化
- [ ] N+1查询已避免
- [ ] 连接池配置合理
- [ ] 缓存策略合理

### 5.3 并发支持
- [ ] 6-8人同时在线无问题
- [ ] 无竞态条件
- [ ] 锁机制正确

## 6. 代码质量验证

### 6.1 代码规范
- [ ] 命名规范一致
- [ ] 代码注释完整
- [ ] 无重复代码
- [ ] 函数单一职责

### 6.2 错误处理
- [ ] 所有异常已捕获
- [ ] 错误日志完整
- [ ] 用户提示友好

## 7. 跑团网站项目特定验证

### 7.1 用户系统
- [ ] GM和玩家角色权限正确
- [ ] 用户数据隔离正确

### 7.2 房间系统
- [ ] 房间人数限制为8人
- [ ] GM权限正确
- [ ] 房间状态管理正确

### 7.3 实时通信
- [ ] WebSocket连接稳定
- [ ] 消息广播正确
- [ ] 断线重连正常

### 7.4 骰子系统
- [ ] 骰子随机性正确
- [ ] 骰子结果记录完整
- [ ] 修饰值计算正确

## 8. 验证工具

- [ ] ESLint/Prettier 代码检查通过
- [ ] 无安全漏洞扫描警告
- [ ] 单元测试覆盖核心功能
- [ ] 集成测试验证主要流程
