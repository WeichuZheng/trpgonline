# Checkpoint Save Skill

本 Skill 用于在跑团网站项目中保存当前工作进度。

## 使用场景

- 完成一个阶段性任务后
- 长时间离开前
- 开始可能影响当前进度的操作前
- 每天工作结束时
- 任何需要保存当前状态的时候

## 执行步骤

### 1. 确认保存内容
- 检查所有已修改的文件
- 确认需要保存的变更
- 排除不需要提交的文件（如本地配置文件）

### 2. 记录当前状态
- 记录当前正在处理的任务
- 记录已完成的工作
- 记录待完成的工作
- 记录遇到的问题或阻塞点

### 3. 保存进度
- 如果使用版本控制：提交当前变更
- 创建清晰的提交信息，说明进度
- 推送到远程仓库（如果需要）

### 4. 记录检查点信息
检查点信息应包含：
- 检查点编号/名称
- 创建时间
- 当前任务描述
- 已完成的工作
- 待完成的工作
- 重要笔记

## 跑团网站项目特定记录

### 项目背景
- 项目名称：trpgonline（在线跑团网站）
- 用户规模：支持 6-8 人同时访问
- 角色区分：GM（主持人）和玩家

### 记录模板
```
检查点：[编号/名称]
时间：[创建时间]
任务：[当前正在处理的任务]

已完成：
- [已完成的工作项 1]
- [已完成的工作项 2]

待完成：
- [待完成的工作项 1]
- [待完成的工作项 2]

问题/备注：
- [遇到的问题或需要注意的事项]
```

### 关键模块状态
根据当前开发的模块，记录相关状态：
- 用户认证模块
- 房间管理模块
- 实时消息模块
- 角色卡牌模块
- 骰子系统模块

## 注意事项

- 创建检查点后，通知用户检查点已保存
- 如果存在敏感信息（如密钥），不要将其包含在提交中
- 遵循项目的安全规则（参考 `.claude/rules/no-secrets.md`）
- 验证保存后文件的完整性
