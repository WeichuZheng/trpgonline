# Resume Work Skill

本 Skill 用于在跑团网站项目中恢复之前保存的工作进度。

## 使用场景

- 重新开始之前中断的工作
- 切换到另一个任务后需要返回
- 从之前的检查点继续开发
- 重新打开项目时恢复状态

## 执行步骤

### 1. 查找检查点
- 查看之前的检查点记录
- 确认最近一次保存的状态
- 了解待完成的工作

### 2. 检查项目状态
- 确认当前工作目录
- 检查代码是否有未提交的变更
- 确认开发环境正常

### 3. 恢复上下文
根据检查点记录，恢复：
- 当前正在处理的任务
- 已完成的工作
- 待完成的工作
- 之前遇到的问题

### 4. 验证环境
- 检查依赖是否完整
- 确认数据库连接正常
- 验证之前的工作是否可以继续

### 5. 继续工作
- 与用户确认恢复的进度
- 开始继续之前的任务
- 如有问题，先解决再继续

## 跑团网站项目特定恢复

### 项目背景
- 项目名称：trpgonline（在线跑团网站）
- 用户规模：支持 6-8 人同时访问
- 角色区分：GM（主持人）和玩家

### 恢复检查清单
恢复时请确认以下模块的状态：
- [ ] 用户认证模块（注册/登录）
- [ ] 房间管理模块（创建/加入/退出）
- [ ] 实时消息模块（WebSocket通信）
- [ ] 角色卡牌模块（CRUD操作）
- [ ] 骰子系统（投掷/结果展示）

### 常见恢复场景

**场景1：继续功能开发**
- 确认之前开发的功能点
- 检查相关验收标准
- 继续实现剩余功能

**场景2：修复Bug**
- 查看Bug描述
- 定位相关代码
- 编写测试验证修复

**场景3：处理之前的问题**
- 回顾问题描述
- 分析可能的解决方案
- 尝试解决并验证

## 注意事项

- 恢复前确认之前的工作已完成保存
- 如果远程有更新，先同步再继续
- 恢复后先进行基本测试确保环境正常
- 如需切换到其他分支，先保存当前状态
