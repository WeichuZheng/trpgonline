# Start New Project Skill

本 Skill 用于在跑团网站项目中开始新的开发任务。

## 使用场景

当需要开始一个新功能开发、修复bug或进行任何新的开发工作时使用此skill。

## 执行步骤

### 1. 确认项目状态
- 检查当前工作目录是否在正确的项目路径
- 确认项目名称为 trpgonline（在线跑团网站）
- 检查是否有未完成的工作需要保存

### 2. 理解任务
- 明确新任务的目标和需求
- 如果任务不清晰，先与用户确认
- 识别任务涉及的范围（前端/后端/数据库）

### 3. 检查环境
- 确认开发环境已正确配置
- 检查依赖是否已安装
- 确认数据库连接配置

### 4. 创建分支（如使用Git）
- 为新功能创建独立的分支
- 分支命名规范：feature/功能名、bugfix/问题描述
- 切换到新分支开始工作

### 5. 任务规划
- 分析任务所需的代码变更
- 识别需要修改的文件
- 评估工作量并向用户确认

## 跑团网站项目特定要求

### 项目背景
- 在线跑团(TRPG)网站
- 支持 6-8 人同时访问
- 区分 GM（主持人）和玩家两种角色

### 技术栈考虑
- 前端：Web界面，需支持多用户实时交互
- 后端：API服务，处理房间管理、消息通信
- 数据库：存储用户、房间、角色、消息等数据

### 验收标准参考
- 参考 `.claude/evaluation/frontend-criteria.md` 了解前端验收标准
- 参考 `.claude/evaluation/api-criteria.md` 了解API验收标准
- 参考 `.claude/evaluation/database-criteria.md` 了解数据库验收标准

## 注意事项

- 开始新任务前，确保之前的工作已保存到检查点
- 如有未完成的重大变更，先创建检查点
- 遵循项目的代码规范和验证规则（参考 `.claude/rules/`）
